Este proyecto implementa la función minimo() en Python, incluye pruebas con pytest (caso correcto, límite y error) y un pipeline de GitHub Actions que corre automáticamente en cada push a main.

¿Qué hace?

Devuelve el menor número de un iterable. Si hay errores (vacío, None o no-numéricos), lanza excepciones.

¿Qué demuestra?

Ciclo DevOps: pipeline verde ✅, falla provocada 🔴, corrección ✅.

Ideal para entrega

Incluye documentación, estructura, pruebas y automatización lista para capturas en GitHub Actions.

Cómo usar la función

Importa minimo desde minimo.py y pásale una lista (o iterable) de números.

# app.py (ejemplo rápido)
from minimo import minimo

numeros = [5, 2, 9, 1]
resultado = minimo(numeros)

print(resultado)  # 1
  • Si la lista está vacía → ValueError
  • Si contiene un valor no numérico → TypeError
  • Si se pasa None → TypeError

Pruebas (pytest)

Ejecuta todas las pruebas desde la raíz del proyecto:

python3 -m pytest -v

Ejecutar pruebas individuales:

python3 -m pytest -v tests/test_minimo.py::test_minimo_caso_correcto
python3 -m pytest -v tests/test_minimo.py::test_minimo_caso_limite_un_solo_elemento
python3 -m pytest -v tests/test_minimo.py::test_minimo_caso_error_lista_vacia

CI con GitHub Actions

El pipeline corre automáticamente en cada push a main usando Python 3.10, instala dependencias y ejecuta pytest. 

# .github/workflows/ci.yml (resumen)
name: CI
on:
  push:
    branches: ["main"]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.10"
      - run: pip install -r requirements.txt
      - run: pytest -q
  • ✅ Pipeline verde: tests pasan
  • 🔴 Forzar fallo: cambia un esperado en un test
  • ✅ Reparar: vuelve a corregir el test y repite push
